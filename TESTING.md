# 本地测试指南

本指南提供了如何在本地环境中测试博客项目的详细步骤。

## 测试前准备

### 1. 安装依赖

确保您的系统已安装 Python 3.6 或更高版本，然后运行以下命令安装项目依赖：

```bash
pip install -r requirements.txt
```

### 2. 初始化数据库

首次运行项目前，需要初始化数据库。运行以下命令：

```bash
python init_db.py
```

## 运行项目

### 开发模式运行

使用以下命令启动开发服务器：

```bash
python run.py
```

服务器将在 `http://localhost:5000` 上运行。

## 测试功能

### 1. 登录/注册功能测试

#### 注册功能
1. 访问 `http://localhost:5000/register`
2. 输入用户名（2-8个字符）、密码（6-20个字符）和确认密码
3. 点击"注册"按钮
4. 验证是否自动登录并跳转到首页

#### 登录功能
1. 访问 `http://localhost:5000/login`
2. 输入管理员账号：`adminVD`，密码：`@Wtdlxwsmlm329`
3. 点击"登录"按钮
4. 验证是否登录成功并跳转到首页
5. 测试登录失败的情况（输入错误的用户名或密码），验证是否显示错误提示

### 2. 文件夹分类功能测试

1. 登录后，访问首页 `http://localhost:5000`
2. 验证页面上方是否显示"全部"、"欢迎"、"日常"、"项目"四个文件夹导航
3. 点击不同的文件夹，验证是否只显示对应文件夹中的文章

### 3. 新建博客功能测试

1. 登录后，点击导航栏中的"新建文章"
2. 输入标题、选择文件夹、填写摘要和Markdown内容
3. 点击"保存"按钮
4. 验证文章是否成功创建，并显示在首页对应文件夹中

### 4. 编辑博客功能测试

1. 登录后，在首页找到一篇文章，点击"编辑"按钮
2. 修改文章内容，包括标题、文件夹、摘要等
3. 点击"保存"按钮
4. 验证文章是否成功更新

### 5. 删除博客功能测试

1. 登录后，在首页找到一篇文章，点击"删除"按钮
2. 在确认对话框中点击"确定"
3. 验证文章是否成功删除，不再显示在首页

## 测试注意事项

1. 确保本地环境已安装所有依赖包
2. 测试前初始化数据库，确保数据结构正确
3. 测试时注意观察浏览器控制台是否有错误信息
4. 测试完成后，可以通过 Ctrl+C 停止开发服务器

## 常见问题排查

1. **数据库初始化失败**：检查是否已安装 SQLAlchemy，确保数据库文件有写入权限
2. **依赖安装失败**：尝试使用 `pip install --upgrade pip` 升级 pip 后再安装依赖
3. **服务器启动失败**：检查端口 5000 是否被占用，或修改 `run.py` 中的端口配置
4. **功能测试失败**：检查浏览器控制台错误信息，查看服务器日志输出
